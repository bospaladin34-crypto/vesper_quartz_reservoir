# External Partner Briefing: System Classification & Compliance

## Overview
The VESPER Braid SDK and Santos Protocol implement a **deterministic, physics‑driven computational model**.  
It is not a cryptographic system and does not rely on randomness, entropy, or key material.

## Key Points for Legal, Compliance, and Partner Teams

### 1. Non‑Cryptographic Nature
The system contains:
- no encryption or decryption  
- no key generation or key exchange  
- no hashing or message authentication  
- no random number generation  
- no entropy pools  

### 2. Deterministic Architecture
All operations are:
- deterministic  
- reversible  
- geometry‑constrained  
- parity‑preserving  

This aligns the system with **scientific/engineering software**, not cryptographic software.

### 3. Regulatory Position
Because the system is non‑cryptographic:
- cryptographic export controls do **not** apply  
- NIST cryptographic standards do **not** apply  
- cryptographic compliance frameworks do **not** apply  

### 4. Applicable Standards
Only general deterministic system‑engineering standards apply, such as:
- NIST SP 800‑160 (Systems Security Engineering) — deterministic sections only  

### 5. Risk Profile
Relevant risks:
- operational misuse  
- integration errors  
- configuration issues  

Irrelevant risks:
- key compromise  
- randomness failure  
- cryptographic algorithm weakness  

### 6. Recommended Classification
> **Deterministic Physics‑Driven Computational Framework**  
> (Non‑cryptographic, non‑probabilistic)

This classification should be used in:
- legal reviews  
- compliance filings  
- partner documentation  
- automated scanner overrides
